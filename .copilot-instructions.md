# Tilda - Car Booking System

## Domain

**What:** Shared car booking for a closed group
**Pattern:** Day-based reservations (one day = one booking unit)
**Conflicts:** Allow overlapping requests (resolve manually/socially)
**Cancellation:** Soft delete with full audit trail

## Users

- Closed group (known users)
- Auth via Oak (method TBD - social, email, or password)
- Proxy may pass auth headers

## Tech Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Backend | Clojure | Core language |
| Reactivity | Datastar | No frontend code, server-driven UI |
| Database | XTDB | Native Clojure, bitemporal (audit trail), learning opportunity |
| Auth | Oak (Gaiwan) | Clojure-native, flexible auth methods |
| HTTP | Ring + Reitit | Standard, composable |
| HTML | Hiccup | Data-driven templates |

## Architecture Principles

1. **Config over code** - resource details, booking rules in `config.edn`
2. **One instance = one resource** - keep scope tight
3. **Curl-friendly API** - JSON responses, simple endpoints, GNU philosophy
4. **Web UI with Datastar** - reactive without writing JS
5. **Domain logic decoupled from DB** - custom dates for conflicts, XTDB valid-time for audit only
6. **Two-table model** - requests (expressions of interest) → bookings (confirmed)
7. **Existence = state** - booking exists = confirmed, deleted = cancelled (history preserved)
8. **Pure deciders** - conflict resolution logic is testable without DB

## API Design

```
GET  /                    # Web UI home
GET  /calendar            # Calendar view

# Requests
GET  /requests            # List pending requests
POST /requests            # Submit a request for a slot
GET  /requests/:id        # Get one

# Resolution
POST /requests/resolve    # Resolve competing requests (admin/cron)

# Bookings
GET  /bookings            # List confirmed bookings
GET  /bookings/:id        # Get one
DELETE /bookings/:id      # Cancel (delete, history preserved)
GET  /bookings/:id/history # View change history (XTDB time-travel)

# Availability
GET  /available?date=     # Check availability
GET  /resource            # Resource info from config
```

## Data Model

```clojure
;; Request - expression of interest
{:xt/id #uuid "..."
 :tenant-name "Flo"
 :start-date #inst "2026-03-01T00:00:00Z"
 :end-date #inst "2026-03-03T00:00:00Z"
 :priority 0
 :status :pending  ; :pending | :accepted | :rejected
 :requested-at #inst "..."}

;; Booking - confirmed reservation (created via resolve-slot!)
{:xt/id #uuid "..."
 :request-id #uuid "..."  ; link to winning request
 :tenant-name "Flo"
 :start-date #inst "2026-03-01T00:00:00Z"
 :end-date #inst "2026-03-03T00:00:00Z"}
 ;; No status needed - exists = confirmed, deleted = cancelled
 ;; Use booking-history for audit trail

;; User comes from Oak session
{:user/id "user-123"
 :user/email "alice@example.com"
 :user/name "Alice"}
```

## Domain Logic (tilda.booking)

```clojure
;; Requests
(create-request! node {:tenant-name start-date end-date priority})
(get-request node id)
(pending-requests node)
(find-conflicting-requests node start end)

;; Resolution (request → booking)
(resolve-slot! node requests decider-fn)  ; atomic: winner→booking, losers→rejected

;; Bookings
(get-booking node id)
(all-bookings node)
(find-conflicts node start end)           ; finds existing bookings
(cancel-booking! node id)                 ; deletes, history preserved
(booking-history node id)                 ; XTDB time-travel

;; Deciders (pure functions)
(decide-first-come-first-serve requests)
(decide-by-priority requests)
(decide-random-lottery requests)
```

## Curl Examples (Target API)

```bash
# Submit a request
curl -X POST localhost:8080/requests \
  -H "Content-Type: application/json" \
  -d '{"start-date": "2026-03-01", "end-date": "2026-03-03", "notes": "Day trip"}'

# List pending requests
curl localhost:8080/requests

# Resolve requests for a slot (picks winner)
curl -X POST localhost:8080/requests/resolve \
  -H "Content-Type: application/json" \
  -d '{"start-date": "2026-03-01", "end-date": "2026-03-03", "decider": "first-come"}'

# List bookings
curl localhost:8080/bookings

# Cancel booking
curl -X DELETE localhost:8080/bookings/abc-123

# View history of a booking (proves it existed)
curl localhost:8080/bookings/abc-123/history

# Available days this month
curl "localhost:8080/available?month=2026-03"
```
